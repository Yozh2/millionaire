Постановка задачи регрессии, как от неё прийти к нейронным сетям, чем нейросети схожи и отличаются от полиномиальной регрессии.
Устройство нейрона. Как обучить нейрон. Функции активации и их различия.

### Questions:

1. Что такое машинное обучение в самом общем смысле?

- Жёсткое программирование всех правил вручную
- Автоматический подбор параметров модели по данным
- **Процесс, при котором модель учится на данных находить закономерности**
- Случайный перебор ответов без обучения

2. Какое направление машинного обучения использует размеченные данные с правильными ответами?

- Обучение без учителя
- Обучение с подкреплением
- **Обучение с учителем**
- Самоорганизующееся обучение без примеров

3. Что обычно является целью задачи регрессии?

- Выбрать один из нескольких классов
- **Предсказать непрерывное числовое значение**
- Сгенерировать изображение из текста
- Найти структуру кластеров без меток

4. Из чего состоит базовый искусственный нейрон?

- Только из функции потерь и датасета
- Из двух независимых моделей
- **Из входов, весов, смещения и функции активации**
- Из свёрточного фильтра и пулинга

5. Зачем нужна функция активации в нейроне?

- Чтобы уменьшить размер датасета
- Чтобы сделать данные размеченными
- **Чтобы добавить нелинейность в модель**
- Чтобы заменить оптимизатор

6. Какая функция активации классически использовалась в перцептроне?

- Softmax
- ReLU
- **Пороговая функция**
- LogSoftmax

7. Что делает градиентный спуск?

- Увеличивает ошибку на каждом шаге
- Выбирает случайный класс
- **Итеративно уменьшает значение функции потерь**
- Строит дерево решений

8. Что обычно означает learning rate?

- Размер датасета
- Количество слоёв сети
- **Шаг обновления параметров при оптимизации**
- Вероятность переобучения

9. Почему backpropagation важен?

- Он заменяет датасет
- Он делает сеть линейной
- **Он позволяет эффективно считать градиенты для всех весов**
- Он убирает необходимость в функции потерь

10. Что помогает бороться с проблемой локальных минимумов и шумных градиентов?

- Случайное удаление слоёв
- **Метод моментов**
- One-hot encoding
- Увеличение числа выходов

11. Что такое выпуклая функция ошибки?

- Функция с бесконечным числом минимумов без структуры
- **Функция, у которой локальный минимум совпадает с глобальным**
- Функция только с дискретными значениями
- Функция без производной

12. Что чаще всего сравнивают с нейросетью при переходе от регрессии?

- Таблицу истинности
- **Полиномиальную регрессию**
- Дерево каталогов
- Алфавитную сортировку

13. Что обычно означает one-hot encoding?

- Нормализацию чисел до нуля
- **Представление класса в виде вектора с одним единичным элементом**
- Перевод текстов в аудио
- Удаление выбросов из данных

14. Для чего делят данные на train, validation и test?

- Чтобы увеличить число параметров
- **Чтобы обучать, настраивать и честно оценивать модель**
- Чтобы убрать все ошибки из датасета
- Чтобы сеть работала без обучения

15. Что делает перцептрон в задаче классификации?

- Строит изображение по тексту
- **Разделяет пространство признаков гиперплоскостью**
- Запоминает каждый пример отдельно
- Подбирает только нелинейные фильтры

16. Почему от классического перцептрона отказались как от универсального решения?

- Он работал только с изображениями
- **Он не решал нелинейно разделимые задачи и был слишком ограничен**
- Он требовал GPU
- Он не умел использовать веса

17. Чем ADALINE исторически был важен?

- Он был первой GAN-моделью
- **Он использовал градиентный подход к обучению линейного нейрона**
- Он был первым трансформером
- Он решал только задачи кластеризации

18. Что означает использование скрытых слоёв в нейросети?

- Сеть становится обязательно деревом
- **Модель может строить более сложные нелинейные представления**
- Сеть перестаёт обучаться
- Ошибка всегда равна нулю

19. Что обычно происходит при слишком большом learning rate?

- Обучение всегда становится точнее
- **Оптимизация может начать расходиться или прыгать вокруг минимума**
- Датасет автоматически очищается
- Сеть перестаёт считать градиенты

20. Какой общий смысл у backpropagation и gradient descent вместе?

- Один генерирует данные, другой сортирует их
- **Один считает градиенты, другой использует их для обновления весов**
- Один убирает слои, другой добавляет слои
- Один кодирует классы, другой декодирует их

### Questions:

1. Что является основной идеей нейросети?

- Хранить данные только в таблице
- **Комбинировать простые вычислительные элементы в многоуровневую модель**
- Искать только линейные зависимости
- Делать выбор без обучения

2. Для чего обычно применяют CNN?

- Для сортировки чисел по возрастанию
- **Для обработки данных с локальной пространственной структурой, например изображений**
- Для генерации случайных таблиц
- Для хранения текста в словаре

3. Что отличает RNN от обычной полносвязной сети?

- У неё нет параметров
- **Она учитывает последовательность и внутреннее состояние во времени**
- Она работает только с картинками
- Она не обучается градиентами

4. В чём главная проблема затухающих градиентов?

- Ошибка становится комплексной
- **Градиенты становятся слишком малы на длинных цепочках слоёв или шагов времени**
- Модель перестаёт принимать входные данные
- Данные перестают быть числовыми

5. Что помогает LSTM и GRU лучше обычной RNN?

- Случайное удаление всех весов
- **Механизмы памяти и управления потоком информации**
- Только использование ReLU
- Замена входов на изображения

6. Что делает автокодировщик?

- Классифицирует только два класса
- **Сжимает данные в латентное представление и восстанавливает их обратно**
- Сортирует изображения по яркости
- Увеличивает размерность без обучения

7. Чем VAE отличается от обычного AE?

- У него нет декодера
- **Он учит вероятностное латентное пространство**
- Он не использует нейросети
- Он работает только с текстом

8. Для чего часто используют GAN?

- Для линейной регрессии
- **Для генерации реалистичных новых данных**
- Для подсчёта медианы
- Для сжатия таблиц в CSV

9. Что такое свёртка в CNN по сути?

- Случайный выбор пикселей
- **Скользящее применение фильтра к локальным областям входа**
- Обратное сортирование матрицы
- Удаление всех признаков

10. Почему ResNet стал важным этапом в эволюции CNN?

- Он убрал все свёртки
- **Он ввёл residual connections, облегчающие обучение очень глубоких сетей**
- Он работал без входных данных
- Он заменил изображения текстом

11. Что обычно делает pooling в CNN?

- Увеличивает число каналов без причины
- **Снижает пространственное разрешение и делает признаки компактнее**
- Преобразует классы в one-hot
- Убирает необходимость в обучении

12. Что такое латентное пространство?

- Список всех файлов на диске
- **Сжатое внутреннее представление данных в модели**
- Таблица с названиями классов
- Случайный шум без структуры

13. Какой фреймворк чаще рекомендуют для исследовательской работы и прототипирования нейросетей в этой теме?

- Только Excel
- **PyTorch**
- Только HTML
- Только Bash

14. Что хранит тензор в PyTorch?

- Только текстовые метки
- **Многомерный массив данных с поддержкой операций на CPU/GPU**
- Только изображение в PNG
- Только журнал ошибок

15. Зачем нужен граф вычислений?

- Чтобы скрыть формулы от пользователя
- **Чтобы автоматически считать производные при обучении**
- Чтобы сжать датасет
- Чтобы убрать слои из модели

16. Почему важно делить данные на train/validation/test?

- Чтобы ускорить интернет
- **Чтобы не переоценить качество модели и корректно её настроить**
- Чтобы обучать только на test
- Чтобы убрать шум из изображений навсегда

17. Что делает нормализация данных?

- Переводит ответы в один класс
- **Приводит признаки к сопоставимому масштабу**
- Увеличивает шум в датасете
- Заменяет функцию потерь

18. Какой из перечисленных наборов относится к известным датасетам изображений?

- IMDb, WikiText, LibriSpeech
- **MNIST, CIFAR-10, ImageNet**
- COCO, EuroParl, SQuAD
- LibriSpeech, GLUE, WMT

19. Что обычно делает MLP?

- Обрабатывает последовательности с памятью
- **Работает с векторными признаками через полносвязные слои**
- Считает только свёртки изображений
- Генерирует звук без входа

20. В чём общий смысл обучения любой нейросети в PyTorch?

- Ручная настройка каждого выхода без градиентов
- **Чередование прямого прохода, вычисления потерь, backpropagation и шага оптимизатора**
- Только загрузка датасета в память
- Только экспорт модели в ONNX

1. Зачем вообще нужен научный поиск перед написанием работы?

- Чтобы выбрать случайную тему
- **Чтобы понять контекст, пробелы и существующие решения**
- Чтобы заменить чтение экспериментами
- Чтобы избежать цитирования источников

2. Что лучше всего помогает быстро оценить релевантность статьи на первом проходе?

- Сразу читать приложение
- **Название, аннотацию, ключевые слова и выводы**
- Только номер страниц
- Только список литературы

3. Для чего полезен менеджер библиографии вроде Zotero?

- Для компиляции кода
- **Для хранения источников, метаданных и цитирования**
- Для рисования графиков в браузере
- Для замены научной статьи презентацией

4. Чем Zotero часто удобен для исследователя?

- Он работает только без интернета
- **Он позволяет собирать, тегировать и цитировать источники в одном месте**
- Он заменяет чтение статей
- Он пишет статью автоматически без правок

5. Что обычно является слабым местом ручного ведения списка литературы?

- Наличие аннотаций
- **Высокий риск ошибок и трудоёмкость обновления**
- Слишком много тегов
- Невозможность сохранять PDF

6. Какую задачу решает Zettelkasten?

- Сжатие изображений
- **Связывание коротких заметок в сеть идей**
- Подсчёт формул в статье
- Генерацию ссылок на случайные сайты

7. Что лучше всего отражает принцип Zettelkasten?

- Одна длинная заметка на всё

- Только скриншоты без текста
- Полный отказ от ссылок между заметками

8. Чем полезен Obsidian для исследовательской работы?

- Только как генератор DOI

9. Что обычно содержится в хорошей заметке по статье?

- Только название файла
- **Краткая суть, ключевые идеи, ограничения и ссылка на источник**
- Только цитата из введения
- Только список авторов

10. Какой источник лучше использовать для первичного поиска англоязычных научных статей?

11. Что полезно проверять у статьи помимо самой идеи?

- Цвет обложки

- Только шрифт таблиц
- Только благодарности

17. Какой подход полезен для оценки качества результатов статьи?

- Сравнить только заголовок
- **Проверить метрики, базовые линии и корректность эксперимента**
- Доверять одному красивому графику
- Игнорировать раздел experiments

18. Почему важно хранить DOI, URL или BibTeX-данные сразу?

- Чтобы увеличить число страниц
- **Чтобы потом быстро вставлять корректные ссылки и цитаты**
- Чтобы скрыть источник от читателя
- Чтобы уменьшить размер заметки до нуля

19. Что полезно делать после чтения хорошей статьи?

- Закрыть файл и забыть о нём
- **Сформулировать выводы своими словами и связать их с другими заметками**
- Удалить все примечания
- Переписать абзацы без понимания

20. Что является признаком хорошей исследовательской системы заметок?

- Все записи изолированы друг от друга
- **Быстрый поиск, связность идей и возможность повторного использования**
- Только цветные стикеры без текста
- Отсутствие ссылок на первоисточники
