# Рабочие процессы

Этот документ — навигация по каноническим workflow проекта `millionaire`.
Подробные правила разнесены по профильным файлам, чтобы не смешивать
агентский контекст, разработку, контент игр и экспорт артефактов.

## Как выбрать документ

- `docs/agent-workflow.md` — правила локального контекста агентов,
  документационных изменений и больших архитектурных планов.
- `docs/development-workflow.md` — разработка фич, quality gate, git hooks и
  browser smoke для UI-изменений.
- `docs/game-authoring.md` — добавление игр, кампаний, вопросов, тем и
  game-specific ассетов.
- `docs/export-workflow.md` — standalone game builds, hub bundles, Docker
  images, Electron desktop artifacts и offline smoke.
- `docs/asset-loading-strategy.md` — фактическая runtime-загрузка ассетов,
  манифестов, аудио и изображений.
- `docs/loading-trace.md` — воспроизводимые waterfall traces и выводы по
  загрузке.

## Документационный принцип

Документация в `docs/` — часть продукта. Обновляй её в той же задаче, если
меняется:

- архитектура `engine`, `hub`, `pages`, `games` или `public`;
- контракт `GameConfig`, `campaign`, `question`, `lifeline`,
  `prizeLadder`, `rewardKind`;
- правила загрузки ассетов, манифестов, аудио или изображений;
- команды запуска, сборки, экспорта, деплоя или генерации;
- пользовательский flow выбора игры, старта кампании, прохождения вопросов,
  завершения игры или звукового согласия;
- правила добавления новых игр и кампаний;
- структура каталогов.

Если появляется новый канонический документ, добавь его в
`docs/repository-layout.md`. Если он нужен новым разработчикам, добавь ссылку
и в `README.md`.

## Базовый порядок задачи

1. Прочитать обязательные документы из `AGENTS.md`.
2. Создать агентский локальный контекст по `docs/agent-workflow.md`.
3. Проверить `git status --short` и не перетирать чужие изменения.
4. Найти существующие паттерны через `rg` и ближайшие тесты.
5. Сформулировать минимальный план в `current-task.md`.
6. Внести scoped-изменения.
7. Обновить профильную документацию, если изменилось поведение или структура.
8. Запустить релевантные проверки из `docs/development-workflow.md`.
9. Обновить `current-task.md` и `worklog.md`.
